# Claw 错题管理系统 - 独立桌面应用（含 NAS 跨网段同步）

## 功能概述

| 功能 | 说明 |
|------|------|
| 文件队列管理 | 拖拽/选择文件/文件夹，图片预处理 |
| 批量 OCR 处理 | 上传→提交→轮询→结果 四阶段流水线 |
| 报告管理 | Markdown 预览、ZIP 下载、删除 |
| 处理历史 | 记录查询，行内查看/下载 |
| 系统配置 | API 密钥、服务器参数管理 |
| **NAS 跨网段同步** | **SMB NAS 数据中转与双向同步** ✨ |

## 快速启动

```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 启动后端服务（需要先启动）
cd ../backend
python main.py

# 3. 启动桌面应用
cd ../standalone
python main.py
# 或双击 run.bat
```

## NAS 跨网段同步说明

### 架构

```
网段 A                     NAS (192.168.0.79)              网段 B
┌─────────────────┐        ┌──────────────────┐        ┌─────────────────┐
│ 桌面应用实例 #1  │──推送──→│  SMB 共享 (share) │←──拉取──│ 桌面应用实例 #2  │
│                 │←──拉取──│  /claw_sync/     │──推送──→│                 │
│ output/reports/ │        │  ├─ reports/     │        │ output/reports/ │
│ 本地处理历史    │        │  ├─ history/     │        │ 本地处理历史    │
└─────────────────┘        │  └─ shared_config│        └─────────────────┘
                           └──────────────────┘
```

### 工作模式

1. **自动同步**（默认启用）：每次 OCR 处理完成后，自动将生成的报告推送到 NAS
2. **手动推送/拉取**：在「NAS 同步」标签页手动执行单项或全量同步
3. **离线缓存**：当 NAS 不可达时，数据暂存本地 `local_cache/` 目录，恢复连接后自动刷新
4. **远程浏览**：可直接查看 NAS 上的报告列表，选择性拉取或删除

### NAS 配置

在「系统配置」→「NAS 跨网段同步配置」中设置：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| NAS 地址 | 192.168.0.79 | SMB 共享服务器 IP |
| 共享名 | share | SMB 共享目录名 |
| 账号 | maker | SMB 认证用户名 |
| 密码 | maker | SMB 认证密码 |
| 同步根目录 | claw_sync | NAS 上的同步数据根目录 |
| 挂载盘符 | (空) | 可选，如 `Z:` 使用盘符访问 |
| 自动同步 | 启用 | 处理完成后自动推送报告 |

### 重连策略

- 网络中断时自动进入重连循环
- 采用**指数退避**策略：2s → 4s → 8s → 16s → 32s → 64s → 120s（上限）
- 最多重试 5 次，失败后进入离线模式
- 每 30 秒进行一次健康检查，发现恢复后自动重连并刷新离线缓存

### 文件校验

上传文件到 NAS 后，通过**文件大小比对**验证拷贝完整性，确保数据可靠传输。

## 技术栈

- **PyQt6**：桌面 UI 框架
- **httpx**：HTTP 异步客户端
- **smb_sync**：自研 SMB 同步模块（基于 Windows net use + UNC 路径）
- **PaddleOCR API**：百度 AI Studio 文档结构化分析

## 注意事项

1. 需要先启动 `backend/main.py` 后端服务
2. NAS 同步功能依赖 Windows SMB 客户端，无需额外安装第三方 SMB 库
3. 跨网段使用时请确保 NAS IP (192.168.0.79) 在目标网段中可达
4. 多实例并行时请避免同时对同一报告执行写操作
